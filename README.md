# KSS RNG Manipulation RTA 2
- RTAでバトルウィンドウズ戦の乱数調整を行うためのツール。洞窟大作戦は未実装。

## 乱数調整の仕組み
- 最初に乱数の仕組みについて簡単に説明する。本作のランダム要素は、乱数が「0～255」のうちどの範囲にあるかで決まる。例えば「カービィが着地したときに出る星の方向」は、乱数が0～31なら上、32～63なら右上という具合だ。(※乱数は実際には0～65535の値を取るが、乱数の判定に使われるのは下位8ビットのため0～255になる。判定方法も実際は何通りの値を得たいかを掛けてから8ビット右シフトした値を使うので、星の向きはその値が0なら上、1なら右上となる。)
- 乱数はゲーム起動時に特定の値から始まり、乱数が使われるたびに特定の計算式によって更新される。これは実質的にあらかじめ用意された数値のリストを辿っているようなものだ。このリストのどこに位置しているか(乱数位置)を特定することで、意図的に乱数を進めて理想的な結果を引き出せる。(※乱数は0～65535のうち、2つの数値を除いた65534回のサイクルで1周する)
- 乱数の範囲は有限で並び方も決まっているから、充分な観測値があれば乱数位置を特定できる。本ツールでは、あらかじめバトルウィンドウズに辿り着いたときの乱数位置が少なくともどの範囲にあるかを入力しておき、実際のバトルウィンドウズ戦の最初に星を3回か4回出してその向きを入力することで乱数を絞り込み、どう行動すれば理想的な結果になるかを計算する。

## 乱数範囲の探し方
- 人によってプレイスタイルは違い、道中での乱数の進み方が異なるため、実際に何度かプレイして自分の適性範囲を見つけることをお勧めする
  1. 設定から到着時の乱数位置の表示をオンにした状態で、乱数位置範囲をデフォルトの範囲より広めに設定しておく。
  2. ソフトリセットからバトルウィンドウズ戦までを通しプレイする。
  3. 乱数が完全に特定できるまで星の向きを入力し、到着時の乱数位置を確認する。
> [!Warning]
> - ソフトリセット時にタイトル画面で余計な乱数を消費しないようにする必要がある。HAL研究所のロゴが出たらA/B/X/Yのどれかを押し、ロゴが消えたら他のA/B/X/Yのどれかを押し続けることで、タイトル画面を最速でスキップできる(日本語版でのみ可能)。これを行わないと、タイトルロゴが画面に表示されている間、毎フレーム乱数が進む。
> - 探索する乱数範囲が狭いほど、より簡単な行動で良い乱数を引きやすくなる。特に星の回数が3の場合は、魔法使いで解決不可能なパターンにならないように、乱数位置の最大値を3376以下にすることを推奨する。

## 乱数調整のための行動
- 星の向きを入力し終えると、各ターンで「先制を取るため」と「レッドドラゴンにガードさせるため」の行動が表示される。
  - 待機: 文字通り何もせず待つ。乱数は消費されない。
  - 星: ジャンプして着地して星を出す。2消費。
  - スライディング: スライディングを出す。6消費。
  - 鬼殺し: 鬼殺し火炎ハンマーを出して空振りする。14消費。
- ダッシュは最初の煙は2消費し、以降のダッシュ継続中の煙は、再入力でも自動発生でも1消費する。ただし一瞬だけダッシュすれば1だけ消費できる
  - 短ダッシュ: 一瞬だけダッシュする。1消費。
  - ダッシュ: ダッシュして次の煙が出る前にやめる。2消費。
  - 長ダッシュ: 2回目の煙が出るまで走るか、再入力してもう一度煙を出す。3消費。

## 分岐
- 本ツールでは、乱数の候補が複数あっても共通して問題ない行動を表示するようにしてあるが、どうしても解決できないパターンもある。その場合はコピーの元によって行動を分岐させる。
- 表の「分岐」の列に対象となるコピーの元と分岐先の行動が表示される。実際にゲームで表示されたコピーの元と一致した場合は「分岐」の列の行動をし、そうでなければ元の列の行動を続ける。

## Fastモード
- 難易度選択でEasyを選んだ場合、コピーの元の判定が終わるのを待ってから鬼殺しをする必要がある。一方、Fastモードはこれを待たずに鬼殺しをすることができる。
- 敵に最速で鬼殺しをする場合、以下の順で乱数が消費される。
  1. 溜め中の煙による消費
  2. ハードヒット判定
  3. コピーの元判定
  4. ハードヒットによる消費
  5. 終了時の煙による消費
- すぐに鬼殺しをするとそのタイミングによって、「溜め中の煙によっていくつ消費された段階で先制判定が行われる」かが変わる。これ自体は複数のパターンに適合するように調整すれば解決するが、厄介なのは、コピーの元判定とハードヒット判定の順番が変わることだ。コピーの元判定は場合によっていくつ乱数が消費されるか異なり、これによって撃破後の乱数が異なる場合がある。
- 出現したコピーの元によって行動を分岐させるという手もあるが、それに気を取られるくらいなら、そもそも乱数の順番が変わらないように最速で鬼殺しをすることに集中したほうがRTA的には良いだろう。
- 猶予は4F。先制判定タイミングは2パターンあり、両方に適合する行動が表示される。
- タイミングを測るにはボスの体力ゲージが有用だ。悪魔の騎士は「ENEMY」の「E」と「N」の間あたりから、「E」の中央付近まで。レッドドラゴンは「E」の直後から、白い枠線と水色のピクセルの間あたりまで。

## 魔法使いのFastモード
- 魔法使いのFastモードではテキストが流れている最中にコピーチェンジを行う。(※Easyモードではテキストが流れる前に行う前提)
- スライディングは最速でやっても6消費する前に先制判定が始まってしまい、5消費になる。一瞬遅らせることで4消費にもできる。
  - 最速スライディング: 最速で出して5消費。
  - 準速スライディング: 一瞬遅らせて4消費。
- 調整したうえですぐに鬼殺しをする時間がないので、鬼殺しの溜め中の煙による消費を逆に利用して調整を行う。
- タイミングによって消費数が異なるため、以下の4つのパターンを定義している。(8消費することも可能だが恐らく猶予1Fなので除外)
  - Fast1(6消費/猶予3F？): 恐らく4F目からハードヒット判定がコピーの元判定の後ろにズレてしまうため、実質猶予は3F。
  - Fast2(4消費/猶予4F)
  - Fast3(2消費/猶予4F)
  - Fast4(0消費/猶予4F以上？): 先制判定の後に煙による消費を行う。タイミングが遅いと煙の消費がコピーの元判定に干渉する。
- ツール上の設定として、Fastモードには2種類用意している。
  - Aggressive Fast: Fast1～4での調整を優先する。
  - Conservative Fast: 他の行動を優先する。
- Switch版での体感値だが、Fast1は水色ピクセルに達する直前、Fast2は水色ピクセルに達した所、Fast3は星の先端に達する直前、Fast4は星の先端の直後を狙う。 

## ツールの改善内容
- 従来では敵一体ずつ解決していたが、これは最善ではない。例えば、3ターン何もせず最後のターンで7回ダッシュするより、毎ターン1回ダッシュずつするほうが簡単だ。本ツールでは、難易度の高い魔法使いを最優先しつつ、残りのターンは全ての組み合わせから合計難易度が最も低いものが選ばれる。単純に探索範囲も広がるため、良い乱数を見つける確率も上がる。
- 乱数調整のために行う行動も改善した。従来では魔法使いでスライディングの6消費が間に合わないことが考慮されていなかったが、このツールでは、最速スライディングと準速スライディングを追加した。また、悪魔の騎士の後の調整で鬼殺しを使わないように変更。使える行動の選択肢も増えたため、調整のしやすさが大幅に向上した。
- 従来のコードは非常に難解だったため、乱数計算処理を作り直した。しかし、コピーの元の再抽選処理の結果が違ったので、実際にゲームをプレイして検証したところ、作り直したもののほうが正しいことが分かった。コピーの元の種類の乱数値の範囲についても従来は一部間違っていたが、新しい方式では何通りの値を得たいかに基づいて適切に変換されるので問題ない。(この変換は非常に一般的なので、恐らくゲームの実際のコードでも同じ処理)
